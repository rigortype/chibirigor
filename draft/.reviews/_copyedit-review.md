# 日本語 校正・校閲レビュー（前編＋共通前付け）

> 日本語に堪能な校正・校閲者として振る舞う独立サブエージェント（opus）による、**日本語表現の
> 観点だけ**の校閲。技術的正否・定着技術用語（例外を投げる／untyped に倒す／degrade／differ 置換）
> には踏み込まない方針。カジュアルさは残しつつ技術書の品位へ。

## 総評
日本語の地力は高い。比喩は身体感覚で一貫（育てる・縫う・倒す・丸める・呑む）。致命的破綻なし。
癖は 3 つ：(A) 慣用句の誤用が一点複数箇所、(B) 軽すぎる起こし言葉、(C) 太字・誇張の過剰。

## すでに適用した修正（明確な誤り）
- **「腑に落とす」→「腑に落ちる」**（自動詞の他動詞化の誤用）：preface・part0。文を「〜生まれる
  ―それが①の概念で腑に落ちること」に組み替え。
- **「体に入れる」→「頭に入れる／身につける」**（意味不明な比喩）：part0 §0.3・part4 地ならし。
- **「実はこれ、+ に限りません」→「これは + に限った話ではありません」**（AI 調の軽さ）：part2。
- **「さて本題。」→「さて、ここからが本題です。」**（口語ブログ調の体言止め）：part6。
- **「いつものパターン」→「恒例の枠組み」**：part1。
- **part4 まとめの「Part 3 の Hash を merge」→「不変 Scope に with_local」**（narrow を Scope クラス
  API に直した内容と不整合だった＝表記ゆれ指摘から発見）。「素朴な Hash 止まり」→「素朴な Scope 止まり」。

## 著者判断・留保（未適用。横断的・好みに関わる）
- **太字・傍点の過密**（横断）：ほぼ全段落に `**…**` があり「ここぞ」（never frighten／:maybe は
  罰しない 等）が埋もれる。**1 段落 1 強調**を目安に間引くと品位が上がる。日本語の誤りではない
  ので著者判断。
- **誇張形容詞の反復**：「決定的に違う」「正反対」「いちばん大事」が章をまたいで複数。章ごとに
  一度ずつへ絞ると効果が戻る。
- **「いよいよ」の常套句**：章予告でほぼ毎回。半減を推奨。
- **「氾濫」＋「だらけ」**（part5）：漢語と口語の同義反復気味。どちらかへ。
- **「differ 置換」**（part7）：一般読者に像が結びにくい。Rigor 界隈の定訳なら初出に注記、そうで
  なければ「ふるまいを変えない差し替え」等へ。＝**技術用語か要確認（留保）**。
- **「合言葉」vs「標語」**（part0／part9）：同じ "never frighten working code" の呼び分け。意図的
  （slogan のニュアンス差）なら可、ゆれなら統一。
