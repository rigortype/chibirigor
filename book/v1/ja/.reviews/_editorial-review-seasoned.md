## レンズ別の所見

### 1. 整合（改訂の接続・相互参照の実在）
- **相互参照は健全**。`前編 §1-3`＝little/part1 §1-3「check ― 矛盾を見つける（でも止まらない）」（"そっと通す"／"脅かさない"を含む）、`前編 §9-2`＝little/part9「わざと見逃す 4 箇所」を正しく指す（L2 改訂「§7-4 の前編§1-3 引用是正」は反映済み・実在確認）。付録参照（a1・a2-3/a2-4/a2-6・a4）もすべて実在アンカー。glossary の「前編 P◯／後編 P◯」併記も後編の章割当（P1=双方向・P2=部分型・P6=FactStore 等）と一致。
- **挿入注記・gloss はおおむね地に足が付いている**。§7-5 の `consistent(A,B)` gloss（記号衝突回避）、§5-4a の「抽象実行」明示、§5-3 の「let 多相には踏み込まない／Part 3 で扱った」言い換えは、いずれも前後の本文から自然に呼ばれており浮いていない。
- ただし **announce の間引きが中途半端**（指摘 1）。L3 の主目的「§1-3 入口と §7-4 出口の二重 announce を一本化」に対し、Part 6 末予告と §7-7 まとめが「ブックエンド」を温存しているため、結果的に*三重*になっている。接続が切れてはいないが、狙いと逆に密度が上がっている箇所。

### 2. 学習設計・章配分
- **坂は機能している**：1 双方向（地図）→2 部分型/変性→3 ジェネリクス→4 再帰型→（理論の構造ブロック）→5 推論→6 FactStore→（推論/フロー）→7 健全性（頂点）→8 橋。README §「構造 → 推論・フロー → 理論の頂点 → 橋」の宣言どおりで、各章末「次章」予告が次のステップへ手を引く。
- **章末体裁は統一**：全 8 章が「まとめ（箇条書き or 表）＋演習 3〜5 問＋次章予告」を備える。演習は手を動かす設計（subst.rb/unification.rb 等のトレース）で粒度も揃う。Part 8 は「次に読む ADR」へ送る最終章として体裁が適切。
- **図の配置**：5 図は山場（双方向・格子・S-Arrow・μ fold/unfold・6 バケツ）に的確に置かれ、キャプション様式統一（指摘 5 の alt 1 件を除く）。

### 3. 「前編の◯◯は実は△△」フォーマットの一貫
- README が定型を明示的に謳う割に、**開幕文の起点が前編／後編前章／目的提示の 3 通りに割れる**（指摘 2）。本文中盤では各章とも前編の対応箇所（P7・P6・P8・P9 等）を必ず拾うので*回収は果たしている*が、リードの第一印象で定型が立つ章（1・2・6）と立たない章（3・4・5・7）が混在する。README の謳い文句と表面を一致させる最小整形を推奨。

- **frontmatter 健全**：全 8 章＋ README に title/description、sidebar.order が 11〜18 で連番、README は order:2。体裁の崩れなし。
- **README**：章立て表の「起点（前編）」列は本文の回収先と整合（Part4=P6＝有限構造の対比、Part6=P3・P5＝Scope/narrow など、本文で確かに受けている）。「動く設計スケッチ」節の examples 列挙（subtype/subst/mu_typeeq/unification/fact_invalidation）は seasoned/examples/ の実在ファイルと一致。
