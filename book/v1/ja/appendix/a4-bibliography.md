# 付録 a4 ― 参考書・ADR 対応早見表

> **この付録は「任意の道しるべ」です。必読ではありません。** 本書（前編・後編）は、
> 参考書を 1 冊も開かなくても単独で完結します。ここは、各章の脇に散らばっていた
> 「この話は『しくみ』の何章・TAPL の何章にあたるか」「本文のこの緊張は Rigor のどの ADR が
> 解いているか」という対応情報を、一覧で引けるよう 1 つにまとめた**参照表**です。必要な所
> だけ拾ってください。

## a4-1. 書誌

### 参考書（2 冊）

- **『しくみ』** … 遠藤侑介『型システムのしくみ ― TypeScript で実装しながら学ぶ型と
  プログラミング言語』（ラムダノート）。TAPL のエッセンスを*やさしく蒸留*した入門書で、
  型注釈付きのミニ言語に対する*チェッカー*を TypeScript で作ります。注釈を前提にする点で、
  推論を土台にする chibirigor とちょうど裏表の関係。日本語のみ。
- **TAPL** … Benjamin C. Pierce『型システム入門 ― プログラミング言語と型の理論』
  （オーム社。原著 *Types and Programming Languages*, MIT Press）。本格的な教科書で、各話題に
  本式の証明があります。英語で読むなら原著がそのまま参照先になります。

### 道しるべの濃淡（前編と後編で違う）

- **前編 The Little chibirigor** … 参考書は**任意**。各章は参考書ゼロで完結します。『しくみ』が
  *やさしい裏面*として最も近く、対応章を脇に添えてあります。
- **後編 The Seasoned chibirigor** … **TAPL を一次の道しるべ**に据えます（変性・再帰型・System F・
  健全性は TAPL の章立てに沿って進みます）。『しくみ』は要所で併走しますが、後編が踏み込む
  領域（本物の型推論・FactStore）には対応章がありません。

### Rigor の ADR / spec

- **ADR**（Architecture Decision Record）… Rigor の設計判断記録。リポジトリの `docs/adr/`。
  「なぜ open class を許すか」「なぜ Ractor でなく fork か」といった判断が*物語*として残ります。
- **内部仕様（spec）** … `inference-engine.md` 等。ADR が「なぜ」なら spec は「どう」を書きます。

> 型システムそのものの対応章は『しくみ』/TAPL に、Rigor 固有の*工学*の判断は ADR にあります。
> 後編 Part 8 が「最小版 → 実用ツール」の橋として ADR への入口をまとめており、a4-4 はその表を引きます。

## a4-5. この付録の使い方

- **型理論をもう一段覗きたい** → a4-2／a4-3 で対応章を引き、手元の『しくみ』/TAPL を開く。
  どちらも持っていなくても本書は完結します。
- **本物の Rigor を読みたい** → a4-4 から、いちばん気になる行の ADR を 1 本開く。ADR-0 →
  ADR-4 が全体の地図です。
- **本文に戻る** → 各章末の 1 行ポインタからここへ来た場合、表で位置を確かめたら章へ戻って
  ください。早見表は道しるべであって、本筋ではありません。
