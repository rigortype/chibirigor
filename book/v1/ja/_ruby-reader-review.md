# L2 伝・Ruby/Railsジュニア読者 再査読（2026-06-09／改修後）

**読者像**：Ruby で入門し Rails でアプリは作れるジュニア。`String`/`Integer`/`Array`/`Hash`/`Data` は実物として理解、`class`・`nil`・`if`・例外・ブロックは日常。一方で**他言語の型宣言・型注釈のメンタルモデルが無い**。「型」を「クラス」と素朴に同一視している。TypeScript 構文（`x: number`、`A | B`）も RBS 記法も初見。

**観点**：「型≒クラス」の像しか持たない読者に、本文だけで通じるか。(a) 型とクラスのズレを断りなく使う、(b) 型注釈/静的型付けという行為を既知として進める、(c) TS/RBS など別記法を足場扱いする箇所を、引っかかった瞬間に申告。過剰な平易化はしない ― 文脈で無理なく学べる新概念は飛躍ではない。直すのは「Ruby だけの読者がそこで本当に詰まって先に進めない」真の段差だけ。

**重大度**：BLOCK＝詰まり先へ進めない／FRICTION＝引っかかるが文脈で回復可能／nitpick。

---

## 総評（先に結論）

**改修後の本書は、Ruby だけの読者にとって非常によく配慮されている。** 私の素朴な「型＝クラス」像が壊れる最初の地点（Part 1「型は `Integer` のようなクラスそのものではなく、型を表すデータ」）に、明示的な“頭の切り替え”の一文が置かれていて、ここで一度きちんと足場が組み直される。`Data.define` という私の既知の道具で全型キャリアが作られているのも終始効いていて、「型って結局 `Data` のオブジェクトなんだ」という安心感が通読の土台になった。

**BLOCK 級（先へ進めない真の段差）はほぼ無い。** ただし、初見の私が一瞬「これは何を言っているのか」と止まった FRICTION が数点ある。多くは TS/RBS/他言語の用語が*説明より一歩早く*出てくる、または別記法（RBS の `->`、TS の `unknown`/`any`）が断りなく置かれる箇所。文脈や直後の説明で回復はできるが、回復に一拍要る。以下、章ごとに記す。

総じて「Rigor を学ぶ Ruby 経験者の本」として狙ったレベルは保てている。下げる必要はない。下記の FRICTION のうち優先度が高いのは **Part 8 の RBS 記法の初出**（私が初めて「型を書いた記法」に出会う場所）と **Part 0 の `Const[1]` 初出**で、ここだけは一手当てると体験が目に見えて良くなる。
