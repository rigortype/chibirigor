## 共有の作法（既存 8 点に揃える）

既存図 `figures/svg/little-0-1.svg`・`little-2-1.svg`・`little-5-1.svg` の流儀。**新規もこれを踏襲**：

- **ファイル名・番号**：`図 N-1` ＝ `figures/svg/little-N-1.svg`。今回は `little-4-1.svg`／`little-6-1.svg`／
  `little-7-1.svg`。
- **viewBox**：幅 540 を基準（高さは内容に応じて 300〜360 程度）。`role="img"` ＋ `aria-label` ＋ `<title>`。
- **配色は CSS 変数＋16 進フォールバック**で（ダーク/ライト両対応のため必ず変数経由）：

  | 用途 | 変数（フォールバック） |
  |---|---|
  | 既定の線・文字 | `var(--rigor-ink,#0E1116)` |
  | 補助の小ラベル | `var(--rigor-ink-3,#585F6B)` |
  | 塗り（判定ボックス等） | `var(--rigor-paper-2,#EFEBE0)` |
  | **`:yes`／OK** | `var(--rigor-success,#2E6F3F)`（緑） |
  | **`:no`／不足・不一致** | `var(--rigor-error,#B5302B)`（赤） |
  | **`:maybe`／untyped＝わからない** | `var(--rigor-type-untyped,#6E7480)`（グレー） |
  | 結果・強調ボックス | `var(--rigor-accent,#7A2A2A)`（臙脂） |
  | 参考の補注 | `var(--rigor-info,#2D5C9E)`（青） |

- **フォント**：ラベルは `'Inter',system-ui,sans-serif`、**コード・型表記**（`1 | "a"`、`{name:}`、
  `:yes` 等）は `'JetBrains Mono',ui-monospace,monospace`。
- **矢印**：`marker-end` の三角（既存 `little-5-1.svg` の `<marker>` 定義をそのまま流用可。id は図ごとに
  prefix を変える）。線は `stroke-width:1.5`・`stroke-linejoin/linecap:round`。
- **約物は全角**。型・識別子はバッククォート相当のモノスペースで。丸数字は `①②③`（U+2460 系）。
- 図中に長い散文は置かない。**1 枚 1 メッセージ**。

## 図 7-1（最優先）― `accepts` の三値判定フロー ＋ Union 畳み込み

- **配置**：Part 7 `part7-accepts-and-trinary.md`、§7-3a（Union 畳み込み）の直後。
- **伝える一点**：`accepts(expected, actual)` は **`:yes`／`:no`／`:maybe` の三値**を返す。とりわけ
  **untyped が絡めば `:maybe`（白黒つけない＝脅かさない）**、**Union は全メンバを回して一番弱い答えを採る**。
- **本文と一致させる判定の順序**（§7-3・7-3a のコードそのもの）：
  1. `expected` か `actual` が **`Dynamic`（untyped）→ `:maybe`**（グレー）。
  2. `actual` が **Union → 全メンバを `accepts` にかけ「一番弱い答え」**：1 つでも `:no` なら `:no`／
     `:no` 無く `:maybe` あれば `:maybe`／全部 `:yes` なら `:yes`。
  3. `expected` が Union → 「一番強い答え」（どれか 1 つ `:yes` なら `:yes`）。※副筋。主役は 2。
  4. それ以外 → **`widen`（クラスに丸める）して一致比較** → 一致 `:yes`／不一致 `:no`。
- **レイアウト案**（縦フロー＋右に Union 畳み込みの inset）：

  ```
   入力: accepts(expected, actual)
            │
            ▼
   ◇ どちらかが untyped ? ──はい──▶ [ :maybe ]   ← グレー。注記「わからないものは わからない」
            │ いいえ
            ▼
   ◇ actual が Union ?  ──はい──▶┌─ Union 畳み込み（inset・別ボックス）──────┐
            │ いいえ              │ 全メンバを accepts → 一番弱い答えを採る   │
            ▼                     │  例: [:yes, :yes, :no] → :no   （赤）     │
   widen して クラス一致 ?        │      [:yes, :maybe ]    → :maybe（グレー）│
        ├─ 一致 ─▶ [ :yes ]       │      [:yes, :yes  ]    → :yes （緑）      │
        └─ 不一致 ▶ [ :no ]       └──────────────────────────────────────────┘
  ```

  - 3 つの終端チップ `:yes`／`:no`／`:maybe` は**色で意味を持たせる**（緑・赤・グレー）。本文の
    「この『たぶん』が Rigor がやさしくいられる最大の理由」を視覚化するため、`:maybe` チップを
    一番目立たせてよい（グレーの実線枠＋淡い塗り）。
  - 判定は◇（菱形 or 角丸の決定ボックス、塗り `paper-2`）。
- **キャプション**：`図 7-1　accepts の三値判定 ― untyped は :maybe、Union は一番弱い答え`

## 図 6-1 ― open な HashShape は「不足だけ咎める」

- **配置**：Part 6 `part6-hash-and-tuple.md`、§6-3「open か closed か」の結論段落
  （「余分は許し、不足だけ咎める」）の直後。
- **伝える一点**：期待は **open**＝「**少なくとも** これらのキーがあればよい」。**余分なキーは許し
  （誤検知を出さない）、必要なキーが無いときだけ咎める**。
- **本文と一致させる構図**：期待 `{ name: }`（open）に対し、3 つの実引数を並べて判定 ―
  - `{ name:, admin: }`（余分 `admin` あり）→ **OK**（緑✓）。注記「余分は許す（open）」
  - `{ name: }`（ちょうど）→ **OK**（緑✓）
  - `{ age: }`（`name` 不足）→ **`:no`**（赤✗）。注記「必要な `name` が無い ＝ ここだけ咎める」
- **レイアウト案**：

  ```
        期待（open）              実引数を渡してみる              判定
   ┌─────────────────┐        ┌──────────────────┐
   │  { name: }      │◀───────│ { name:, admin: }│ ───▶  ✓ OK   （余分は許す）
   │  少なくとも     │◀───────│ { name: }        │ ───▶  ✓ OK
   │  name があればよい│◀──╳──│ { age: }         │ ───▶  ✗ :no  （name 不足）
   └─────────────────┘        └──────────────────┘
                  ▲ width subtyping（キーが多い方が部分型）
  ```

  - 緑✓は `success`、赤✗は `error`。「余分は許す」「不足だけ咎める」の 2 つの小ラベルは `ink-3`。
  - 期待ボックスは `accent` 枠で強調。`{...}` はモノスペース。
  - 図 5-1（union のナローイング）と画面密度を揃え、横幅 540 に収める。
- **キャプション**：`図 6-1　open な HashShape ― 余分は許し、不足だけ咎める`

## 図 4-1 ― then／else の型をまとめて Union（図 5-1 と対）

- **配置**：Part 4 `part4-union.md`、§4-1 の `type_of(parse(...)) => 1 | "a"` ブロックの直後
  （`IfNode` の union 化を説明した辺り）。
- **伝える一点**：`if`／三項の型は、**then 節と else 節の型を `union` でまとめたもの**。
  図 5-1（union を分岐で*分ける*＝ナローイング）の**逆向き**＝分岐を*まとめる*、として対にする。
- **本文と一致させる構図**（§4-1 の主例 `x = rand < 0.5 ? 1 : "a"`）：
  - 上に式 `x = rand < 0.5 ? 1 : "a"`。
  - then 節 → `1`（`Const`）、else 節 → `"a"`（`Const`）の 2 箱が下から **1 つの `union` ボックスへ
    収束**。
  - 結果 `1 | "a"`（`accent` 強調、図 5-1 の「両枝を union」ボックスと同じ意匠）。
  - 補注（`ink-3`、小さく）：「else が無ければ偽のとき `nil` ― だから `c ? 1 : nil` も `1 | nil`」。
- **レイアウト案**（5-1 を上下反転した収束形）：

  ```
        x = rand < 0.5 ? 1 : "a"
            │              │
       then 節          else 節
            ▼              ▼
        ┌ 1 ┐          ┌ "a" ┐
        └─┬─┘          └──┬──┘
          └────▶ union ◀──┘
                  │
                  ▼
             [ 1 | "a" ]        ← accent 強調
  ```

  - 収束の 2 本の矢印は `accent` マーカー（5-1 の `flittle51_aA` と同色）。
  - 型箱 `1`・`"a"` はモノスペース、`Const` のままの値リテラルである点が伝わるように。
- **キャプション**：`図 4-1　if／三項の型 ― then と else をまとめて Union（図 5-1 の逆向き）`
